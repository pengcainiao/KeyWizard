# 桌宠素材规格单（哭娃打字宠）

> AI 生成卡通帧时，严格按本规格。不达标会导致动画抖动 / 错位。

## 一、通用硬性要求（每一帧都要满足）

| 项 | 要求 | 原因 |
|----|------|------|
| 背景 | **透明** PNG（带 alpha 通道） | 桌宠窗口要透明，露出桌面 |
| 画布尺寸 | **统一 512 × 512**（所有帧一模一样） | 帧之间要对齐 |
| 角色位置/大小 | **每帧里娃的身体、头的位置和大小完全一致**，只变化"手 / 表情" | 否则切帧时娃会跳动 |
| 画风 | 所有帧**同一种卡通画风、同一个角色**（用"同一角色"功能或 img2img 锁定） | 保证是"同一个娃" |
| 还原度 | 寸头、肉脸颊、皱眉大哭神态 ≥ 80% 像照片里的娃 | 这是核心要求 |
| 朝向 | 正面 / 微侧，所有帧一致 | 一致性 |
| 命名 | 严格用下面的文件名 | 程序按名字加载 |

## 二、需要的帧（最小可用集，先做这 4 帧）

放到 `assets/sprites/` 目录：

| 文件名 | 状态 | 画面 |
|--------|------|------|
| `idle.png` | 待机·抽噎 | 娃垂眼、嘟嘴、轻微委屈，双手放下 |
| `cry.png` | 大哭·飙泪 | 皱眉、张大嘴、飙眼泪（还原照片表情），双手放下 |
| `tap_left.png` | 左拳捶键 | 表情同 idle/微哭，**左手小拳头抬起**，右手放下 |
| `tap_right.png` | 右拳捶键 | 表情同 idle/微哭，**右手小拳头抬起**，左手放下 |

> 关键：`tap_left` / `tap_right` 和 `idle` 最好是**同一张图只改手的位置**，这样切换最顺。

## 三、可选增强帧（之后想要再做）

| 文件名 | 状态 |
|--------|------|
| `sleep.png` | 长时间没打字 → 睡着 |
| `cry2.png` | 哭得更凶的第二帧（和 cry 交替做"抽动"） |

## 四、AI 生图建议

1. **先生成 `idle`**，定下角色长相和画风。
2. 用"**同一角色 / 参考图 / img2img**"功能，基于 idle 改出其它帧——只改表情和手，**别改身体位置和大小**。
3. 生成后若背景不是透明，用抠图工具（如 remove.bg / 即梦抠图）去背景，导出透明 PNG。
4. 最后确认 4 张都是 512×512、透明、娃的位置一致。

## 五、交付方式

把 4 个 PNG 放进 `assets/sprites/`，告诉我"放好了"，我就接进 Ebiten 让它动起来。
